RESELL TRACKER — CHANGELOG
==========================

v1.9.0 — 2026-03-11
--------------------
ADDED
  - Discord tab — new dedicated nav tab (admin) separating Discord alerts
    from website monitors.
  - Discord keyword alerts — app auto-starts the selfbot on launch and listens
    for keyword matches in monitored Discord channels.
  - Alert History log — every Discord keyword alert is logged in the Discord
    tab with channel, keyword, message preview, and timestamp. Click any entry
    to jump to it in the Discord app.
  - ATC auto-open — when a Discord alert contains an embedded ATC link (field
    labeled ATC, Add to Cart, Cop, Buy, or Checkout), the link opens
    automatically in the browser.
  - Size-click ATC on Shopify feed — click any in-stock size in the Live
    Product Feed to instantly ATC that variant (opens store.com/cart/VARIANT:1).
  - Neon styling for Monitors (pink) and Discord (indigo) nav tabs.

FIXED
  - Discord channel IDs (64-bit integers) no longer get corrupted when saving
    keywords — JSON is now updated via text replacement to avoid JS float
    precision loss.
  - Discord alert toast now opens the Discord desktop app instead of the browser.

v1.8.0 — 2026-03-11
--------------------
ADDED
  - Pinned Messages — admin can pin announcements visible to all users on
    the Home tab (e.g. drop reminders, watch-out notices).
  - Release time field — admin sets a drop time in their local timezone;
    all users see it auto-converted to their own local time with timezone
    abbreviation (e.g. 10:00 AM EST / 3:00 PM GMT).
  - Product link field on releases — attach a buy/release page URL that
    opens in the system browser directly from the release card.

CHANGED
  - Home calendar redesigned to match a cleaner reference layout: large
    left accent panel with big day number, centered month title with year
    top-right, nav arrows flanking the grid, bigger cells (42px rows).
  - Stats row (This Month Profit, Inventory Items, In Transit) removed
    from Home — cleaner layout.
  - Pinned Messages section is now aligned to the calendar width.
  - Right releases panel widened for better card display.

FIXED
  - Crash on save release after stats row was removed from Home
    (null textContent error on missing DOM elements).
  - Release date circles (orange/today, blue/release) now properly
    centered within their grid columns.

v1.7.0 — 2026-03-10
--------------------
ADDED
  - Home tab — new default landing page with a full release calendar.
  - Shared release calendar — admin pushes upcoming releases (toys, sneakers,
    collectibles, etc.) that all users see instantly on their Home tab.
  - Calendar highlights: orange = today, blue = release dates.
  - Release cards show product image, name, date, retail price, and notes.
  - Stats row on Home: this month's profit, inventory count, packages in transit.
  - Releases stored in a shared PostgreSQL database on Railway — data persists
    forever and is the same for every user.

v1.5.1 — 2026-03-10
--------------------
SECURITY
  - Auth verification now happens server-side on every app launch — removing a
    user's Discord subscriber role takes effect immediately on next open.
  - Discord OAuth client secret moved off the app entirely and onto a private
    backend server (Railway). The secret is no longer bundled in the .exe.
  - Login now issues a signed JWT (RS256) — tokens cannot be forged without
    the server's private key.
  - JWT is bound to the device it was issued on — auth files cannot be shared
    between machines.

v1.5.0 — 2026-03-10
--------------------
SECURITY
  - Rearchitected Discord authentication to use a server-side auth backend.
  - Client secret removed from the app bundle.
  - JWT-based session with device fingerprinting.

v1.4.0 — 2026-03-10
--------------------
ADDED
  - Discord OAuth2 authentication gate — app shows a login screen on launch.
  - Users must sign in with Discord and have the Subscriber role in your server
    to access the app.
  - Token saved locally and reused on future launches (auto-refreshed when
    near expiry — Discord tokens last 7 days).
  - "Sign Out" button in Settings modal.
  - discord-config.js — single file to configure Client ID, Client Secret,
    Guild ID, and Role ID.

v1.1.0
------
ADDED
  - Settings modal with 17track API key configuration.
  - Package auto-refresh on app load for all non-delivered packages.
  - Carrier auto-detection from tracking number format (UPS, USPS, FedEx,
    DHL, LaserShip, OnTrac, Amazon).
  - "Auto-detected" hint shown when carrier is guessed from tracking number.
  - Package event history toggle (expand/collapse older events per package).
  - Mark as Sold modal — sell inventory items directly into the sales log,
    with partial quantity support.
  - Inventory photo support — pick a local file or paste an image URL.
  - Photo lightbox — click any inventory thumbnail to view full size.
  - Profit preview in Add/Edit Sale and Mark as Sold modals (live calculation).
  - Chart year navigation (prev/next arrows) to browse historical sales data.
  - Sales search, inventory search, and package search.
  - Scrollable sales table (max height with sticky header).
  - External link support — opens carrier tracking page in system browser.

CHANGED
  - Sales stats (Spent, Revenue, Profit) calculated across all sales, not
    just the current year.
  - Package status pipeline uses 5 stages: Ordered → Awaiting Pickup →
    In Transit → Out for Delivery → Delivered.
