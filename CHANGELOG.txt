RESELL TRACKER — CHANGELOG
==========================

v1.6.0 — 2026-03-10
--------------------
ADDED
  - Market Lookup tab — search any item and see live eBay results embedded
    directly inside the app (real browser, no scraping).
  - Active / Sold toggle — switch between active eBay listings and recently
    sold listings with one click. Switching auto-refreshes the results.
  - Neon purple glow border around the eBay browser pane in dark mode.
  - Discord profile picture and username now displayed in the top nav bar
    next to the avatar circle.
  - Settings modal now shows a Discord profile card with avatar, username,
    and a Dashboard button at the top.

FIXED
  - Dark mode topnav cream/tan border replaced with a subtle dark separator.
  - Inventory "Add Product" qty field now shows a "Quantity (# of units)"
    label so users understand what the value represents.
  - Auth check no longer hits the server on every settings open — server
    verify only runs once on app startup.
  - Username falls back to JWT payload if auth.json predates username
    storage (avoids showing blank name after update).

v1.5.3 — 2026-03-10
--------------------
CHANGED
  - Version bump to trigger auto-update delivery for v1.5.2 changes.

v1.5.2 — 2026-03-10
--------------------
ADDED
  - Discord profile picture and username now shown in the top nav bar.
  - Settings modal now displays a Discord profile card with avatar, username,
    and a Dashboard button.

FIXED
  - Dark mode topnav no longer shows a cream/tan border — replaced with a
    subtle dark separator.
  - Inventory "Add Product" modal now shows a "Quantity (# of units)" label
    above the qty field so users understand what the value represents.
  - Auth check no longer hits the server on every settings open — only runs
    once on app startup, preventing login conflicts.

v1.5.1 — 2026-03-10
--------------------
SECURITY
  - Auth verification now happens server-side on every app launch — removing a
    user's Discord subscriber role takes effect immediately on next open.
  - Discord OAuth client secret moved off the app entirely and onto a private
    backend server (Railway). The secret is no longer bundled in the .exe.
  - Login now issues a signed JWT (RS256) — tokens cannot be forged without
    the server's private key.
  - JWT is bound to the device it was issued on — auth files cannot be shared
    between machines.

v1.5.0 — 2026-03-10
--------------------
SECURITY
  - Rearchitected Discord authentication to use a server-side auth backend.
  - Client secret removed from the app bundle.
  - JWT-based session with device fingerprinting.

v1.4.0 — 2026-03-10
--------------------
ADDED
  - Discord OAuth2 authentication gate — app shows a login screen on launch.
  - Users must sign in with Discord and have the Subscriber role in your server
    to access the app.
  - Token saved locally and reused on future launches (auto-refreshed when
    near expiry — Discord tokens last 7 days).
  - "Sign Out" button in Settings modal.
  - discord-config.js — single file to configure Client ID, Client Secret,
    Guild ID, and Role ID.

v1.3.0 — 2026-03-10
--------------------
CHANGED
  - Platform column in sales list now displays the platform's full logo image
    (favicon fetched via Google favicon service at 32px).
  - Fallback colored chip badge shown if logo fails to load or no internet.

v1.2.0 — 2026-03-10
--------------------
ADDED
  - Platform badges in sales list — colored chips with 2-letter abbreviations
    and brand colors for 19 platforms: eBay, StockX, GOAT, Depop, Poshmark,
    Mercari, Facebook Marketplace, Amazon, Grailed, Flight Club, Stadium Goods,
    OfferUp, Craigslist, Hermes, Vestiaire, Tradesy, Vinted, and more.
  - Unknown platforms auto-generate a grey chip from the first 2 letters of
    the platform name.

v1.1.0
------
ADDED
  - Settings modal with 17track API key configuration.
  - Package auto-refresh on app load for all non-delivered packages.
  - Carrier auto-detection from tracking number format (UPS, USPS, FedEx,
    DHL, LaserShip, OnTrac, Amazon).
  - "Auto-detected" hint shown when carrier is guessed from tracking number.
  - Package event history toggle (expand/collapse older events per package).
  - Mark as Sold modal — sell inventory items directly into the sales log,
    with partial quantity support.
  - Inventory photo support — pick a local file or paste an image URL.
  - Photo lightbox — click any inventory thumbnail to view full size.
  - Profit preview in Add/Edit Sale and Mark as Sold modals (live calculation).
  - Chart year navigation (prev/next arrows) to browse historical sales data.
  - Sales search, inventory search, and package search.
  - Scrollable sales table (max height with sticky header).
  - External link support — opens carrier tracking page in system browser.

CHANGED
  - Sales stats (Spent, Revenue, Profit) calculated across all sales, not
    just the current year.
  - Package status pipeline uses 5 stages: Ordered → Awaiting Pickup →
    In Transit → Out for Delivery → Delivered.

v1.0.0
------
ADDED
  - Initial release.
  - Three-tab layout: Sales, Inventory, Packages.
  - Sales table with Platform, Product, Purchase Price, Sale Price, Fees,
    Profit, Date columns.
  - Monthly bar chart showing Spent / Revenue / Profit per month.
  - Stat cards: total Spent, Revenue, Profit.
  - Inventory table with photo thumbnail, product name, quantity, store,
    purchase price, estimated resell, and estimated profit.
  - Package tracking cards with pipeline status visualization.
  - Add / Edit / Delete modals for Sales, Inventory, and Packages.
  - Confirm delete modal.
  - Keyboard shortcuts: Escape to close modals, Ctrl+Enter to save.
  - Top nav with Sales / Inventory / Packages tabs and Settings button.
  - Data persisted locally via Electron IPC (electron-store).
